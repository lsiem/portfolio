## Phase Goal (user story)

**As a** visitor on a capable device, **I want to** encounter one living, identity-bound signature moment in the hero, **so that** the site feels like a system that thinks — while every visitor still gets the fast facts unharmed and every promise is proven on production before launch.

*(Same phase user story as 04-01; this plan delivers the "proven on production before launch" precondition — a healthy, honestly-measured 3D-free baseline. MVP slice: after this plan, the CI gate tells the truth about what ships and the deployed build passes TECH-01, which it does not today.)*

<objective>
Complete launch hardening by making the CI performance gate measure what actually ships and closing the deployed-bundle overage, so the site has a genuinely green 3D-free baseline before the constellation is built on top of it.

Today the CI LHCI job audits `http://localhost:3000` via `pnpm start` and reports `script:size` 177,509 PASS, while the same commit deployed to Vercel serves ~308KB of app JS and fails `script:size` (308,250), LCP (/de 3373ms, /en 3725ms), and perf (0.88) — documented in `.planning/debug/phase3-perf-bundle-gap.md`. The gate is hiding the real payload. Measured injection (~29KB, of which 23KB is the preview-only feedback toolbar) accounts for only part of the delta; ~100KB is real app `/_next` JS the localhost audit does not load.

Purpose: honor D-12 (harden first, build the 3D on a healthy baseline, clean attribution) and D-11 (LCP is a hard gate). 04-01 restored the LCP assertion to `error` against the localhost gate; this plan repoints the gate at a representative target and closes the bundle overage so `error` reflects reality. If the 3D were ever cut, launch hardening is then genuinely complete.

Output: a CI LHCI step that audits the deployed preview URL for /de + /en; a closed eager-bundle overage so the representative gate passes all five TECH-01 assertions; the phase3-perf-bundle-gap debug finding resolved.
</objective>

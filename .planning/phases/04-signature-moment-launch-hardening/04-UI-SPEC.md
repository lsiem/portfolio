# Phase 4 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

Scope: WOW-01 (the ONE identity-bound Signature-3D/WebGL moment in the hero — lazy, capability-gated, architecturally removable) plus launch hardening (D-11 LCP work, which touches font delivery but must not change the visual contract). Source of locked decisions: `04-CONTEXT.md` (D-01…D-14) and the approved `03-UI-SPEC.md` (the standing site-wide contract this phase **extends, never re-litigates**). This contract resolves the "Claude's Discretion" visual items from CONTEXT (constellation geometry, densities, drift/pulse values, tier configs) with concrete, prescriptive values so planning/execution do not re-ask.

**Inheritance rule:** everything in `03-UI-SPEC.md` (approved 2026-07-05) remains binding and unchanged — layout/grid, spacing, typography roles, color budget, motion tokens, reduced-motion contract, mobile contract. This document specifies only (a) the new constellation surface and (b) the constraints launch hardening places on existing visuals. Where this document is silent, `03-UI-SPEC.md` governs.

## Typography

**Inherited unchanged from `03-UI-SPEC.md`:**

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (Geist Sans) | 1.6 |
| Label (mono) | 12px | 500 (Geist Mono), eyebrows `uppercase tracking-[0.25em]` | 1.4 |
| Heading (h3) | 18px | 500 (Geist Sans) | 1.3 |
| Display (hero/case-study H1) | `clamp(2.75rem, 2rem + 5vw, 6rem)` | 700–800 (Bricolage variable axis) | 1.05 |

**Phase-4 additions to the typography contract:**

1. **The constellation renders NO text.** D-02 locks "no labels, no legend" — zero typography inside the canvas, zero i18n surface in 3D. Any node labeling idea is explicitly deferred (see 04-CONTEXT Deferred Ideas).
2. **LCP hardening must be typography-invariant.** The D-11 font levers (subsetting Bricolage to the hero-H1 glyph repertoire, `preload: true`, `adjustFontFallback` metric matching) change bytes and timing, never appearance. Acceptance check for execution: a before/after visual diff of the hero H1 at 320/768/1440 shows no rendered difference (same glyphs, same weight, same metrics after swap). If the subset file misses a glyph used in either locale's H1/nav strings, that is a contract violation (fallback-font flash on real content).
3. **Weight tally unchanged:** 400 + 500 on Geist (the 2-weight UI budget) + the single Display role on Bricolage's variable axis — 3 distinct values total, as documented and passed in Phase 3.

## Color

**Palette inherited unchanged (Phase 3 D-02, locked)** — tokens from `globals.css`, light and dark both intentional:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` (`#fafaf9` light / `#0a0a0a` dark) | Page background; the canvas is transparent (`alpha: true`) so the token background reads through — the scene has no background color of its own |
| Secondary (30%) | `var(--muted)` (`#6b6560` / `#8f8a85`) + `var(--border)` (`#e7e5e4` / `#262626`) | De-emphasized text and hairline structure — **and now the constellation's resting material** (see mapping below) |
| Accent (10%) | `var(--accent)` (`#c2410c` light / `#fb923c` dark) | Signal color only — reserved list below |
| Destructive | not applicable | No destructive actions exist this phase (no forms, no delete flows) |

### Constellation token mapping (prescriptive — no hardcoded hex anywhere in scene code)

Scene materials resolve colors at runtime from the CSS custom properties (`getComputedStyle` on `documentElement`), re-resolved on `data-theme` mutation and `prefers-color-scheme` change (RESEARCH Pattern 5). The scene is part of the 30% secondary layer at rest; only pulses spend accent.

| Scene element | Token | Opacity | Rationale |
|---------------|-------|---------|-----------|
| Node dots (resting) | `var(--muted)` | 0.55 | Quiet presence — reads like the spine ticks / mono annotations, part of the engineered structure |
| Edge hairlines (resting) | `var(--border)` | 0.35 | Faintest layer — same visual weight class as the site's 1px hairline rules |
| Message pulse (traveling dot + short edge-segment glow) | `var(--accent)` | 1.0 → 0 (fades over travel) | The ONE signal-color event (D-03/D-04) — accent means "a message is passing," consistent with accent = signal everywhere else on the site |
| Pointer-proximity illumination | node color lerps `var(--muted)` → `var(--accent)`, max 60% blend at radius center | follows node opacity | Same "magnetic affordance" language as Phase-3 buttons (accent glow at max pull, never at rest) — D-06 |
| Boot-assembly flash | nodes may wake at `var(--foreground)` 0.8 for ≤ 300ms before settling to resting `var(--muted)` | — | The "system coming alive" beat reads as activity, not decoration; no accent spend on entrance |

**Accent reserved for (Phase-3 list, items 1–6, unchanged and still exhaustive) PLUS this phase's single extension:**

7. Constellation message pulses and pointer-proximity node illumination (the two scene behaviors above) — accent in the scene is always **transient event signal**, never a resting fill.

No other element may use the accent color. This list (1–7) is exhaustive, not illustrative. The constellation must never render a permanently accent-colored node or edge.

## Constellation Visual Spec (WOW-01 — D-01…D-04, D-08; discretion values resolved)

The one signature moment: a multi-agent constellation — living node graph, points + hairlines, quietly orchestrated — as an **atmospheric background layer** behind the hero text. Wow = noticing the system is real, not spectacle.

### Geometry & density

| Property | Desktop tier | Mobile tier (capable phones, D-07) |
|----------|-------------|-----------------------------------|
| Node count | 72 | 36 |
| Edge count | ~110 (sparse graph, avg degree ~3; never fully connected) | ~54 |
| Node dot size | 2–3 device px (depth-attenuated within this band) | 2 device px |
| Edge line weight | 1 device px hairline (matches site's 1px rule motif) | 1 device px |
| DPR clamp | `[1, 1.5]` | `[1, 1.25]` |

*(Node/edge counts are the prescriptive defaults within Claude's discretion — the executor may tune ±20% based on profiling, never upward past the point where TBT or the D-08 "atmospheric, not busy" judgment degrades.)*

### Composition (D-08 — headline stays the focal point)

- **Density mask:** node placement density behind the H1/value-prop reading zone (the central reading column) is ≤ 30% of the density at the hero's outer thirds. Denser toward edges, sparser behind copy — enforced in `constellation-data.ts` placement, not post-hoc.
- **Contrast ceiling:** at resting opacity the scene must not reduce the H1's effective contrast below WCAG AA against `var(--background)` — trivially satisfied because dots/lines are muted/border tokens at ≤ 0.55 opacity behind text-sparse zones, but execution must verify in both themes.
- **Hidden structure (D-02):** graph topology derives from ELIA's multi-agent shape — a small orchestrator hub cluster with satellite agent clusters (recommended within discretion; career-graph derivation is the sanctioned alternative). No labels, no legend, no i18n surface.

### Temporal behavior (D-04 — drift + message pulses)

| Behavior | Value |
|----------|-------|
| Node drift | Slow 3D oscillation around home positions; max screen-space velocity ≤ 8 px/s, oscillation period 20–40s per node (randomized). Delta-based in `useFrame` (refresh-rate independent) |
| Pulse frequency | One pulse every 2–5s (randomized, irregular — event-driven feel, never metronomic); at most 2 pulses in flight simultaneously |
| Pulse travel | ~800ms per edge (`--motion-duration-slow` as the reference value), accent dot travels the edge, trailing glow fades to 0 behind it |
| Pointer influence (D-06) | `pointer: fine` only, no-op on touch/keyboard. Influence radius 120 screen px; gentle attraction, max node displacement 10 px; illumination per color table above. Same easing family as magnetic buttons (`--motion-ease-magnetic` character) |

### Entrance & exit choreography (D-05, D-09)

| Moment | Behavior |
|--------|----------|
| Boot-beat entrance (intro still running) | Constellation assembles as the **final beat of the existing hero-intro GSAP timeline**: clusters wake sequentially (nodes fade/scale in, then edges draw), total beat ≈ 1200ms (`--motion-duration-chapter`), `--motion-ease-out`. Appended to the existing timeline — never a parallel choreography |
| Late-load entrance (intro already finished — slow network) | Simple opacity ramp 0 → resting values over 800ms inside `useFrame` — no second GSAP timeline, no replayed intro, no waiting state |
| Scroll-linked exit | Scene recedes (translate −z / scale down ~10%) and dissolves (opacity → 0) mapped to hero exit progress 0 → 1 via the one-way `sceneBridge.scrollProgress`. At progress ≥ 1: `frameloop="never"` (pause-first; unmount only if profiling demands — RESEARCH Pattern 7). GPU work is zero for the rest of the visit |
| Scroll-progress source on touch | ScrollTrigger drives it where the motion stack runs (`pointer: fine`); a passive scroll/IntersectionObserver fallback writes the same bridge value on capable phones (RESEARCH Pitfall 8) — the exit must work in both regimes |

All DOM-side choreography uses the existing Phase-3 motion tokens; no new motion tokens are introduced. Scene-interior values (drift, pulse) live in the scene code and are canvas-internal — they are not DOM motion and do not enter `globals.css`.

### State contract (D-07, D-10 — every state is a designed state)

| State | Visual result |
|-------|---------------|
| Gate closed (weak device, `saveData`, low memory, tier < 2, SwiftShader, `?webgl=off`) | **Phase-3 hero exactly as shipped — nothing missing, no substitute artwork, no placeholder, no message.** This is the complete experience, not a degraded one |
| `prefers-reduced-motion: reduce` | Gate never opens (composes the existing MotionProvider signal). Quiet variant = Phase-3 reduced-motion hero, unchanged |
| Loading (gate passed, chunk in flight) | Nothing visible — no spinner, no skeleton, no shimmer. The static hero IS the loading state; the scene appears via its entrance choreography when ready |
| WebGL context loss | Canvas silently unmounts → Phase-3 hero remains. **Never an error message, never a toast** — the designed degrade path is invisible to the visitor |
| Canvas creation throw | ErrorBoundary renders null → same silent fallback |
| Tab hidden / scrolled past hero | Rendering paused (`frameloop="never"`); resumes on return |

**The 3D gate is NOT the motion gate:** capable touch devices DO get the constellation (D-07); only the pointer-influence feature is `pointer: fine`-gated (D-06). Copying MotionProvider's `pointer: fine` mount condition would exclude every phone — contract violation (RESEARCH Pitfall 7).

### Accessibility

- Canvas layer stays `aria-hidden="true"`, `pointer-events: none` at the slot level (pointer influence reads global pointer position; the canvas never intercepts clicks/focus — WOW-04: nothing may block the fast path).
- No information exists only in the scene (D-02: it is decorative-with-hidden-structure); screen-reader and keyboard experience is byte-identical with and without it.
- No flashing: pulse events are ≤ 2 concurrent, small, and slow — nowhere near WCAG 2.3.1 thresholds.

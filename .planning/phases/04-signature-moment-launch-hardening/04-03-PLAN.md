## Phase Goal (user story)

**As a** visitor on a capable device, **I want to** encounter one living, identity-bound signature moment in the hero, **so that** the site feels like a system that thinks — while every visitor still gets the fast facts unharmed and every promise is proven on production before launch.

*(MVP slice: after this plan, a visitor on a capable device sees a real WebGL scene alive in the hero background for the first time — thin proof-of-life content, replaced by the full constellation in 04-04 — while every other visitor's experience is byte-identical to Phase 3.)*

<objective>
Build the WOW-01 delivery infrastructure as a working vertical slice: the capability gate (D-07), the lazy `next/dynamic` canvas mount into the existing `HeroSceneSlot` seam, the D-10 silent fallback (reduced-motion / weak device / WebGL failure → Phase-3 hero, nothing missing), and the deterministic `?webgl=off|force` override that makes every later audit and test reproducible (RESEARCH Pattern 8). The canvas renders a minimal proof-of-life scene interior (a stub replaced by 04-04's full constellation) so the slice is end-to-end observable.

Purpose: WOW-01 demands "lazy-loaded, capability-gated, nie im Initial-Bundle" — those three properties are THIS plan's contract, provable independent of what the scene draws. Landing them first, on the healthy 04-02 baseline, keeps attribution clean (D-12) and keeps all of Phase 4's 3D code deletable as one directory (`src/components/scene/`).

Output: `src/lib/capability.ts`, `src/components/scene/{hero-scene-gate,constellation-canvas}.tsx`, self-hosted detect-gpu benchmarks under `public/benchmarks/`, the gate wired into the hero via `HeroSceneSlot` children, and `evals/scene.spec.ts` proving mount/no-mount/fallback end-to-end.
</objective>

<objective>
Close the single gap from 04-UAT.md Test 4: on the newest, most capable hardware
(Apple M5 Pro in the verified reproduction — and any GPU newer than detect-gpu's
shipped benchmark snapshot) the WOW-01 hero constellation silently never mounts.

Root cause (diagnosed, not hypothesised — 04-UAT.md Test 4): in
`src/lib/capability.ts`, `decideSceneTier()` excludes `gpu.tier < 2` regardless
of `gpu.type`. For a GPU absent from the benchmark data, detect-gpu returns
`{ tier: 1, type: "FALLBACK" }` — a fallback tier, NOT a measured weak-GPU
verdict — yet the caveat probe on the line above already proved hardware GL.
The gate treats unknown-new hardware as weak hardware and returns "none".

Fix (per the UAT-approved fix direction): treat a `type: "FALLBACK"` result as
"unknown GPU → capable" (the caveat probe already proved real GL), while keeping
the `tier < 2` exclusion for genuinely BENCHMARK-matched weak GPUs and the tier-0
BLOCKLISTED / WEBGL_UNSUPPORTED classes. Every other exclusion (reduced-motion,
?webgl=off, saveData, deviceMemory<4, caveat probe) and the CI-critical
SwiftShader-gets-"none" behaviour are preserved byte-for-byte.

Purpose: the signature moment (WOW-01, D-07) must reach the modern devices most
likely to belong to recruiters and tech-leads — the exact audience the core
value targets — instead of only devices detect-gpu happened to benchmark before
release.
Output: a pure, unit-testable `sceneTierFromGpu` classifier in capability.ts
with the FALLBACK fix, and CI-runnable Playwright assertions pinning the
FALLBACK-mounts / BENCHMARK-weak-excluded / blocklisted-excluded contract.
</objective>

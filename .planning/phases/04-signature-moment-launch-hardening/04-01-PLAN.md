## Phase Goal (user story)

**As a** visitor on a capable device, **I want to** encounter one living, identity-bound signature moment in the hero, **so that** the site feels like a system that thinks — while every visitor still gets the fast facts unharmed and every promise is proven on production before launch.

*(Derived from the outcome-shaped ROADMAP Phase-4 goal + the project core value; invents no scope. All three story slots trace to the goal line and WOW-01.)*

<objective>
Launch hardening — get the hero Largest-Contentful-Paint back under the 2500ms budget on the existing (3D-free) site, then restore the CI Lighthouse LCP assertion from `warn` to `error`. This is D-12's locked "harden first, then build the constellation on a healthy baseline" step and closes the Phase-3 production-LCP deferral (D-11).

Purpose: reclaim the LCP headroom the locked D-03 Bricolage display face consumed (~300ms locally, measured in `layout.tsx`), so the constellation (Plans 02-04) is built on a green baseline and any later regression is cleanly attributable to the 3D layer. If the 3D were ever cut, launch hardening is already complete ("architektonisch jederzeit streichbar").

Output: self-hosted, preloaded (optionally subset) Bricolage delivery via `next/font/local`; the `lighthouserc.json` LCP assertion restored to `error`; local production LHCI green on /de and /en.
</objective>

## Phase Goal (user story)

**As a** visitor on a capable device, **I want to** encounter one living, identity-bound signature moment in the hero, **so that** the site feels like a system that thinks — while every visitor still gets the fast facts unharmed and every promise is proven on production before launch.

*(MVP slice: after this plan, "proven on production before launch" is literal — executable launch tests exist and have run against the deployed site, and the evidence file answers ROADMAP success criteria 3 and 4 with recorded numbers, not claims.)*

<objective>
Verify every Phase-4 promise on the deployed URL: the D-14 agent-assisted external-tester panel (scripted 30-second stopwatch test + reduced-motion walkthrough, owner spot-checks the runs), the TECH-01 launch re-verification (production mobile Lighthouse/CWV with the constellation active, LCP <= 2500ms per D-11), and the D-13 mid-tier Android proxy test with the proxy explicitly documented. Results land in a single evidence document.

Purpose: ROADMAP success criteria 3 and 4 are launch verifications that only count on the production URL — CI lab numbers were already shown to diverge from deployed reality (the 04-02 finding). This plan turns "externe Tester bestehen den Test" into reproducible scripts plus recorded evidence, compliant with AGENTS.md (browser agents run against the production URL and add nothing to the shipped site).

Output: `evals/launch/stopwatch.spec.ts`, `evals/launch/reduced-motion.spec.ts`, a `launch` Playwright project keyed to `LAUNCH_URL`, and `04-LAUNCH-EVIDENCE.md` with all recorded results.
</objective>

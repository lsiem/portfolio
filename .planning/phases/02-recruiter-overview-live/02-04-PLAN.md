<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build fs → OG image | OG routes read local ttf font bytes from node_modules at build |
| content model → JSON-LD script | First-party typed content serialized into an inline ld+json script |
| static site → crawlers/social | Prerendered meta + image bytes served, no runtime third-party call |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-03 | Tampering (XSS) | Person JSON-LD via `dangerouslySetInnerHTML` | medium | mitigate | Data is first-party from the typed content model (no user input); `JSON.stringify` output; fields (name/role/urls/email) cannot contain markup — audited at the call site per react security rules |
| T-02-04 | Tampering | OG route reads ttf from a fixed node_modules path | low | accept | Build-time only, fixed first-party path, no user-controlled input; not a runtime surface |
| T-02-15 | Information Disclosure | OG/JSON-LD expose name/role/socials/email | low | accept | All fields are already public on the page by design; no additional data is revealed |
</threat_model>

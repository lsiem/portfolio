# Phase 2 — UI Design Contract

> Visual and interaction contract for the **functional recruiter layer**. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **SCOPE GUARDRAIL (read first).** This phase ships a **neutral, disciplined, intentional baseline** — layout structure, spacing rhythm, type scale, interaction states, dark-mode mechanics, accessibility, SEO/share cards, and a CV document. It formalizes and extends the **existing interim tokens** already shipped in `src/app/globals.css`. It does **NOT** define the signature visual identity, a new color world, bold typographic identity, decorative/scroll motion, or the immersive experience — **those are Phase 3's charter** ("Design Direction & Immersive Experience"). Every place a decision would commit signature identity is explicitly deferred to Phase 3 with a `→ Phase 3` note. The discipline here is **editorial / Swiss-minimal restraint** using the Geist type + existing orange accent + mono eyebrows already in the codebase.

**Requirements covered:** CONT-02, CONT-03, CONT-04, CONT-05, CONT-06, CONT-07, CONT-08, MODE-01, TECH-01, TECH-03, TECH-04, TECH-05, TECH-08.

## Spacing Scale

Base grid is Tailwind's 4px step. All values below are multiples of 4 (one documented sub-grid exception for chips).

| Token | Value | Usage |
|-------|-------|-------|
| 3xs | 4px (`gap-1`, `pl-1`) | Inline label/date gaps, tightest stacks |
| 2xs | 8px (`gap-2`, chip `px-2`) | Chip padding-x, tag lists, tight groups |
| xs | 12px (`gap-3`) | Role-progression item spacing |
| sm | 16px (`gap-4`, `pl-4`) | Default element spacing, inline link rows, timeline rail indent |
| md | 20px (`gap-5`) | Hero intra-block spacing |
| lg | 24px (`gap-6`, `px-6`) | Column gutter, section internal spacing |
| xl | 32px (`gap-8`) | Career entries, skill-domain groups |
| 2xl | 48px | Reserved for larger intra-section breaks |
| 3xl (section) | 80px (`gap-20`, `py-20`) | Section rhythm (mobile), page padding |
| 4xl (section) | 96px (`scroll-mt-24`) | Sticky-header scroll offset |
| 5xl (section) | 112px (`gap-28`, `py-28`) | Section rhythm (desktop), page padding |

**Exceptions:** chip vertical padding `py-0.5` (2px) — a deliberate sub-grid value for the hairline tech/tag chips; keep it, it is the only sub-4 value. Everything else must land on the 4px grid.

**Rhythm intent (not uniform padding):** hero is airier (`gap-5`/`gap-20`), dense data blocks (career roles, skills) tighten to `gap-2`/`gap-3`. Do not flatten every block to one padding — the scale-contrast between the 80–112px section breaks and the 8–16px intra-block gaps is the primary rhythm signal.

## Typography

Existing usage (Geist Sans + Geist Mono). Scale is intentionally small and editorial: one large display, one lead, one sub-head, one body, one mono meta/label.

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|
| Display (hero name, h1) | 48px mobile (`text-5xl`) → 72px desktop (`sm:text-7xl`) | Semibold 600 | ~1.05 (`tracking-tight`) | The **only** 600 usage. Sole display element per page. |
| Lead (hero role / value prop) | 18px (`text-lg`) → 20px (`sm:text-xl`) | Regular 400 | 1.5 | `text-muted`, `max-w-xl` for measure |
| Sub-heading (h3: org, project, skill-domain) | 18px (`text-lg`) | Medium 500 | 1.3 (`tracking-tight`) | |
| Body | 16px (`base`) | Regular 400 | 1.5 | Default paragraph; `text-muted` for secondary prose, `text-foreground` for emphasis |
| Body-small / role description | 14px (`text-sm`) | Regular 400 | 1.5 | Nested role descriptions |
| Eyebrow / section label (h2) | 12px (`text-xs`) | Regular 400 | 1.4 | **Mono**, `uppercase`, `tracking-[0.25em]`, `text-muted` — the signature disciplined touch already in the build |
| Meta / dates / chips / nav | 12–14px (`text-xs`/`text-sm`) | Regular 400 | 1.4 | **Mono**, `text-muted` |

**Weight discipline:** two weights carry the system — **Regular 400** (body, meta, labels, lead) and **Medium 500** (sub-headings, emphasized names, links). **Semibold 600** is a single reserved exception used only for the hero display name (h1). Do not introduce bold (700) or light (300).

**Font-family split:** Geist Sans for all reading text and headings; **Geist Mono** reserved for structural/technical signals — the eyebrow section labels, dates, tech/tag chips, anchor-nav, and code-like metadata. This mono-for-metadata pairing is the baseline's typographic character; keep it consistent.

**Hero must render immediately (MODE-01 / CONT-02):** name + role + one-sentence value prop are static SSR HTML in the first viewport, never gated behind animation.

**→ Phase 3:** any bolder display treatment, variable-font axes, custom pairing, oversized editorial type, or kinetic/split text is deferred.

## Color

Palette is the **existing interim token set** in `globals.css` — a near-monochrome warm-stone ink system with a single orange accent. Do not introduce new hues in this phase.

| Role | Light | Dark | Usage |
|------|-------|------|-------|
| Dominant (60%) — `--background` | `#fafaf9` (warm stone-50) | `#0a0a0a` | Page surface, the field everything sits on |
| Secondary (30%) — `--foreground` + `--muted` + `--border` | fg `#1c1917`, muted `#6b6560`, border `#e7e5e4` | fg `#ededed`, muted `#8f8a85`, border `#262626` | Ink (headings/body), muted secondary text/meta, hairline borders & chips — the structural monochrome layer |
| Accent (10%) — `--accent` | `#c2410c` (orange-700) | `#fb923c` (orange-400) | Scarce; reserved list below |
| Destructive | — | — | **None.** This phase has no destructive actions (read-only portfolio). No destructive color needed. |

**Accent reserved for (explicit — never "all interactive elements"):**
1. The `.` glyph in the `LS.` logo (existing).
2. **Case Study** links — the primary in-page CTA into deep content (existing `text-accent`).
3. The **email address** link in the contact block — primary contact CTA (existing `text-accent`).
4. The **CV download** button — primary document CTA (new, see CV section).
5. `::selection` background (existing).
6. The **active/in-view anchor-nav item** and current-section indicator (new, subtle).

**Accent must NOT color:** GitHub/LinkedIn/external "Visit ↗" links (these are `text-muted` → hover `text-foreground`), tech/tag chips, body text, borders, the heatmap cells, or the theme/locale controls. Keeping accent to ~10% is what makes the baseline read as intentional rather than decorative.

**Contrast (TECH-03, WCAG AA both themes):**
- Body/foreground on background: passes AA in both themes (near-max contrast).
- `--muted` on `--background` — **verify AA (≥ 4.5:1 for text) at implementation.** Light muted `#6b6560` on `#fafaf9` ≈ 4.9:1 (pass for normal text). Dark muted `#8f8a85` on `#0a0a0a` ≈ 6.9:1 (pass). Muted is used for real body prose, so it must clear 4.5:1, not just 3:1.
- Accent link text on background must clear AA: light `#c2410c` on `#fafaf9` ≈ 4.7:1 (pass); dark `#fb923c` on `#0a0a0a` ≈ 8:1 (pass). Re-verify after any token change.

**→ Phase 3:** the signature color direction (whether the site stays warm-stone monochrome, goes dark-first, introduces a second hue, gradients, or atmosphere) is deferred. Phase 2 ships the mechanism on the existing palettes only.

## Dark Mode Mechanics (TECH-04)

Currently dark mode follows the system only (`@media (prefers-color-scheme: dark)`). This phase adds an explicit **3-state toggle: System / Light / Dark**, persisted, no flash. **[OPEN — D-B: 3-state toggle is the recommended default, owner may reduce to a 2-state light/dark switch.]**

**Token strategy (change from pure media query to attribute-driven with system fallback):**
- Keep `:root { … }` light tokens.
- Keep `@media (prefers-color-scheme: dark) { :root { … } }` as the **System** behavior.
- Add explicit overrides that win over the media query:
  - `:root[data-theme="light"] { …light tokens… }`
  - `:root[data-theme="dark"] { …dark tokens… }`
- In Tailwind 4 CSS-first, register the dark variant so utilities can target it: `@custom-variant dark (&:where([data-theme="dark"], [data-theme="dark"] *));` plus the media-query case for System. (Executor to confirm exact Tailwind 4 syntax against current docs.)

**No-flash inline script** (blocking, in `<head>` before first paint, no external calls — DSGVO-safe):
- Reads `localStorage` key `theme` ∈ `{"system","light","dark"}` (default `"system"` when unset).
- If `light`/`dark`: sets `document.documentElement.dataset.theme` accordingly.
- If `system`/unset: leaves `data-theme` unset so the media query governs.
- Ship as a tiny inline `<script>` (string), not a hydrated component, so it runs before paint. No FOUC.

**Toggle control (in header cluster):**
- Three-state control (segmented control or a cycling button). Real DOM, keyboard operable, `aria-label` "Theme" and per-option `aria-pressed`/`aria-checked` state. Persists selection to `localStorage` and updates `data-theme` live.
- Styled with existing tokens: `border border-border`, mono `text-xs`, active segment gets `text-foreground` (inactive `text-muted`); hover `border-foreground/40`. **Accent is not used here.**
- `<meta name="theme-color">` should track the active theme's `--background` for mobile browser chrome.

**→ Phase 3:** final light-vs-dark art direction (which is "primary", accent shifts, atmosphere) is deferred. Phase 2 ships the working mechanism on today's palettes.

## Interaction States (TECH-03 — specify for EVERY interactive element)

Baseline motion is **restraint**: `transition-colors` on color changes only, ~150ms. No transforms, no magnetic buttons, no reveal animation (→ Phase 3). Compositor-friendly only if any motion is added.

**Global focus (mandatory, both themes):**
- Every interactive element shows a **visible `:focus-visible` ring**: `outline` 2px solid using `--accent` (or `--foreground` where accent would clash), `outline-offset: 2px`. Never remove an outline without an equivalent visible replacement.
- Tab order follows DOM order (already logical). All controls reachable and operable by keyboard.

| Element | Default | Hover | Focus-visible | Active/Current |
|---------|---------|-------|---------------|----------------|
| Muted link (GitHub, LinkedIn, Visit, legal, org) | `text-muted` | `text-foreground` (`transition-colors`) | ring per global rule | — |
| Accent link (Case Study, email) | `text-accent` | `text-foreground` | ring | — |
| Anchor-nav item | `text-muted` mono | `text-foreground` | ring | in-view section → `text-foreground` + optional accent marker |
| Logo `LS.` | fg + accent `.` | `opacity-70` | ring | — |
| Locale switcher | `border-border text-muted` | `border-foreground/40 text-foreground` | ring | — |
| Theme toggle segment | inactive `text-muted` | `border-foreground/40` | ring | active `text-foreground`, `aria-pressed`/`aria-checked` |
| CV download button (primary) | `bg-accent text-background` OR `border-accent text-accent` (see CV) | darken/`text-foreground` per variant | ring (offset so it clears the fill) | — |
| Tech/tag chip | `border-border text-muted` mono | non-interactive (static) | n/a | n/a |

**Reduced motion (TECH-03 / MODE-02 groundwork):** honor `prefers-reduced-motion: reduce` — disable `html { scroll-behavior: smooth }` (wrap in a media query) and any non-essential transition. Baseline content is fully static, so reduced-motion parity is trivially met here; Phase 3 owns the full quiet-variant.

## CV-PDF (CONT-08)

Two deliverables: the **download UI** and the **PDF document's visual contract**. (PDF *generation technique* is a separate research item — the roadmap flags "CV-PDF-Generierung … kein verifiziertes Pattern"; this spec does not choose it.)

**Download UI:**
- **One CV per locale:** DE under `/de`, EN under `/en`. The button downloads the CV matching the current locale.
- Placement: primary affordance in the contact block; optional secondary in the header cluster. **[OPEN — D-C]**
- Filename pattern: `Lasse-Siemoneit-CV-de.pdf` / `Lasse-Siemoneit-CV-en.pdf`.
- Styling: **primary CTA = the one place a filled accent button is allowed.** Recommend `bg-accent text-background`, mono or sans label, `rounded-md`, `px-4 py-2`, with the global focus ring (offset 2px). Alternative disciplined variant: `border border-accent text-accent` outline button. Pick one and use it consistently. Must have an accessible label including format, e.g. `aria-label` "Download CV (PDF, English)".
- Copy: see Copywriting Contract.

**PDF document visual contract (its OWN clean document layout — NOT the web page printed):**
- Format A4, generous margins (~20mm), single or clean two-column (sidebar for contact/skills + main for career/projects) — **[OPEN — D-E: one-column recommended for ATS-friendliness; two-column is a Phase-3-ish polish, defer if uncertain.]**
- Typography: Geist Sans throughout (embed/subset), one accent-orange rule or heading tint permitted, otherwise monochrome ink on white. Body ~10–11pt, headings ~13–16pt, name ~22–24pt.
- Content sourced from the **same content model** (single source of truth): name, role, value prop, career timeline (D-02 arc), selected projects, domain-grouped skills, contact facts (email/GitHub/LinkedIn — **no phone/address**, matching the schema's data-minimization stance).
- Real selectable text (not an image), correct locale, page numbers, generation/"as of" date.
- No third-party runtime calls; generated at build time or on a server route without leaking data externally (DSGVO).

## GitHub Activity (TECH-08)

- **Contribution heatmap**, last ~12 months, placed in the about/contact area.
- **Build-time fetch + ISR daily revalidation** (`revalidate` ~86400s). **No client-side runtime call to GitHub** — DSGVO hard constraint: the shipped page makes no third-party runtime request. Data is baked at build / regenerated server-side on ISR.
- **Neutral styling using existing tokens:** a week×day grid of small rounded cells. Empty/low days use `--border`; intensity ramp uses a **monochrome `--foreground` opacity ladder** (e.g. 4–5 steps: border → foreground/20 → /40 → /70 → foreground). **Do NOT use accent** for the ramp (keeps accent scarce and keeps the grid theme-adaptive in both light/dark). Cells `~11px`, `gap-1`, `rounded-[2px]`.
- Accessible: the grid has an `aria-label` summary (e.g. "GitHub contributions, last 12 months") and a visually-hidden text total; individual cells may carry `title` with date + count but the grid must convey its meaning without hover.
- **Empty / error state:** if the fetch fails or returns nothing at build, the section must degrade gracefully — render a static fallback line (see Copywriting), never a broken grid or a runtime retry to a third party.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Vercel/local env → server code | GITHUB_TOKEN is read only in server-only code at build time |
| GitHub GraphQL → rendered grid | The API response is untrusted external input rendered into the page |
| static site → visitor | Baked contribution data served; no runtime third-party request |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Information Disclosure | GitHub read-only PAT | high | mitigate | Unprefixed `GITHUB_TOKEN` read only in `import "server-only"` github.ts; never `NEXT_PUBLIC_`; `.env*` is gitignored; token is read-only `read:user` scope; verified by the "no NEXT_PUBLIC_ / server-only" acceptance grep |
| T-02-02 | Tampering (Injection) | GitHub GraphQL response rendered into the grid | medium | mitigate | Only numeric `contributionCount`/`date` are consumed to pick a fixed style bucket and optional title text; response is never passed to `dangerouslySetInnerHTML`; React auto-escapes any text; a malformed/hostile payload falls to the null fallback |
| T-02-16 | Denial of Service | GitHub API unavailable at build | low | mitigate | try/catch → null → static fallback line; build never fails on GitHub outage; rate limit is a non-issue at build cadence |
</threat_model>

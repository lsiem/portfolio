<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → DOM attribute | The persisted theme value is read client-side and applied to documentElement before paint |
| inline script → page | A raw `<script>` runs before hydration |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-08 | Tampering (XSS) | no-flash inline `dangerouslySetInnerHTML` script | low | mitigate | Script body is a static string literal with zero interpolation of request/user data; only reads localStorage and mutates documentElement.dataset; no external/network call (DSGVO) |
| T-02-13 | Tampering | localStorage theme value | low | accept | Only "light"/"dark" are honored (else falls through to System); a tampered value at worst selects a valid theme — no trust decision depends on it |
| T-02-14 | Information Disclosure | theme resolution | low | mitigate | No third-party/CDN call to resolve theme or theme-color; all local |
</threat_model>

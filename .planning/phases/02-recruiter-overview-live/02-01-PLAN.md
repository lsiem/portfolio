<objective>
Complete the functional recruiter overview so a hurried visitor finds every fact fast, on the existing single-scroll `/[locale]` page.

**Phase goal (user story):** **As a** hurried recruiter or tech-lead, **I want to** find who Lasse is, what he does, his timeline, skills, and contact/CV on a fast bilingual page, **so that** I can evaluate him in under 30 seconds.

This plan is the anchor vertical slice: it takes the interim overview (which already renders hero/career/projects/skills/contact) and finishes it into a complete, accessible, one-click-contactable recruiter layer. It adds the hero value proposition (CONT-02) — sourced from a new `ContactInfo.valueProp` content-model field so the hero and the Plan 02 CV share one source of truth (CTX-05, resolves REVIEW finding 2) — an About block (CONT-06 text-first per CTX-03, teaser sourced from the existing `about.mdx` frontmatter description, resolves REVIEW finding 3), the persistent header Contact affordance (CONT-07 / D-A), the CV download button in the contact block only (CONT-08 UI half / D-C — the PDF file itself is produced by Plan 02, and MUST exist before cutover; no header CV mirror), an updated anchor-nav (MODE-01 / CTX-01), and the accessibility baseline (TECH-03: focus rings, reduced-motion scroll guard, malformed-date guard in formatMonth). Career-first ordering (CTX-02) and depth-weighted projects (CONT-04) are already correct — preserve them.

**REVIEW coupling note (findings 2 & 4):** 02-01 and 02-02 are a co-dependent Wave-1 vertical-slice pair that ship together in the single Phase-2 PR (CTX-07). This plan AUTHORS `ContactInfo.valueProp` (content model) and SHIPS the CV download button; Plan 02 CONSUMES `valueProp` and PRODUCES the PDF the button targets. They cannot depend on each other (that would be circular), so the coupling is enforced by the shared content-model field + the no-404 cutover gate in 02-07, not by convention.

Purpose: after this plan the site is a complete, deployable, accessible recruiter overview.
Output: finished `/[locale]` overview + header cluster Contact affordance + a11y baseline, plus the failing-then-passing Playwright assertions that prove it.
</objective>

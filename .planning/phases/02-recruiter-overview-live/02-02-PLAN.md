<objective>
Generate the downloadable CV as a build-time static PDF per locale, from the content model — resolving the roadmap-flagged unknown (CTX-05).

**Slice contribution:** this is the generation half of the CV feature; Plan 01 ships the download button. The technique is the RESEARCH §1 recommendation: `@react-pdf/renderer` v4.5.1 run by a standalone `tsx` script chained into `prebuild`, writing `public/Lasse-Siemoneit-CV-{de,en}.pdf`. Geist Sans ttf is embedded from the `geist` npm package (RESEARCH: it ships raw ttf on disk). One-column ATS layout per D-E; contact facts limited to email/GitHub/LinkedIn (schema has no phone/address).

**Dependency note (REVIEW finding 2 fix):** this plan `depends_on: [02-01]` and runs in Wave 2. The dependency is genuine and one-way: the CV's `CvDocument` reads `ContactInfo.valueProp`, a schema field + per-locale values that Plan 01 authors in `content/shared/types.ts` and `content/{de,en}/contact.ts`. Without that field first, Task 2's `tsc --noEmit` and Task 3's `generate:cv` fail deterministically. This plan produces nothing Plan 01 consumes (the CV button href is a static string), so the ordering is strictly 02-02 → 02-01, not a mutual/circular coupling.

Purpose: deliver CONT-08 without any runtime third-party call (DSGVO) and without adding to the client bundle.
Output: two locale PDFs in `public/`, a reusable react-pdf document, and the wired build step.
</objective>
